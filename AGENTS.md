# Runcheck Website Project Notes

## Architecture

- This project is an Astro 6 static site for `https://runcheckapp.com`.
- Cloudflare Pages target: build command `npm run build`, output directory `dist`.
- Keep `astro.config.mjs` static-only: `output: "static"`, `site: "https://runcheckapp.com"`, `integrations: [sitemap()]`, and Tailwind v4 through `@tailwindcss/vite`.
- Do not add `@astrojs/tailwind`, `@astrojs/cloudflare`, SSR adapters, CMS tooling, animation libraries, or UI component libraries for the foundation.
- Article search has one source of truth in `src/data/searchIndex.mjs`: `buildSearchIndex()` creates the static client index and `searchArticles()` filters it on the client.
- English and Finnish article search use the same shared search implementation. The static endpoints are `/articles/search-index.json` and `/fi/artikkelit/hakuindeksi.json`; each must filter the shared `articles` collection by `locale`.
- Finnish articles live under `src/content/articles/fi/<internal-hub>/` and use `locale: "fi"`. Their public URLs use translated hub slugs from `src/data/articlePaths.mjs` and translated hub copy from `src/data/articleTaxonomy.fi.mjs`.
- `scripts/import-runcheck-articles.mjs --locale=fi` imports the numbered Finnish source set from the Obsidian `fi-FI` folder, converts source headings and metadata to collection frontmatter, and creates descriptive Finnish filename slugs without replacing English articles.
- English/Finnish article pairs are matched by `sourceNumber`. Both versions must emit self-referential and reciprocal `hreflang` links through `BaseLayout.astro`; English remains the `x-default` version.
- Published article locales are `en`, `fi`, `sv`, `nb`, `de`, and `da`. Every available translation is paired by `sourceNumber`; detail pages emit reciprocal `hreflang` links only for versions that exist, while English remains `x-default`.
- Swedish, Norwegian Bokmål, German, and Danish article routes are generated by the shared dynamic route family under `src/pages/[locale]/[articles]/`. Locale roots, search-index paths, Open Graph locales, navigation labels, and localized hub paths have one source of truth in `src/data/articleLocales.mjs` and `src/data/articlePaths.mjs`.
- `scripts/import-runcheck-articles.mjs --locale=<locale>` copies translated Obsidian Markdown into `src/content/articles/<locale>/`, derives a localized SEO slug from the translated H1, localizes search tags, deduplicates identical source-number copies, and never deletes source files. Every published locale currently contains source numbers 1–160.
- Regional pricing has one source of truth in `src/data/pricing.mjs`. `src/components/RegionalPrice.astro` renders Free and Pro price data attributes, and `src/layouts/BaseLayout.astro` uses Cloudflare `/cdn-cgi/trace` country detection cached in `sessionStorage` as `runcheck-pricing-country` to apply the matching tier. Do not add a manual country selector or browser-locale based price detection.
- Homepage `SoftwareApplication` JSON-LD pricing offers must be built from `getStructuredOffers()` in `src/data/pricing.mjs` so visible regional prices and `eligibleRegion` structured data stay aligned.
- Analytics IDs and script URLs have one source of truth in `src/data/analytics.mjs`. `src/layouts/BaseLayout.astro` loads the Google tag and Cloudflare Web Analytics beacon once per page head from that module, and `public/_headers` must allow the Google/Cloudflare script origins plus collection endpoints in CSP. Do not duplicate Measurement IDs, Cloudflare Web Analytics tokens, or inline alternate analytics snippets elsewhere.

## Styling

- Tailwind v4 is CSS-first. Design tokens live in `src/styles/global.css` via `@theme`.
- Use self-hosted fonts imported in `src/layouts/BaseLayout.astro`: Manrope and JetBrains Mono from Fontsource packages.
- Avoid scoped CSS and avoid `@apply`. If a future scoped style must use `@apply`, it needs `@reference "tailwindcss";` first.
- Keep dark-only page defaults in `src/styles/global.css`.

## Site Structure

- `src/layouts/BaseLayout.astro` owns `Header`, `<slot />`, `Footer`, global CSS, self-hosted fonts, canonical metadata, sitemap link, Open Graph, and Twitter metadata.
- `src/pages/index.astro` imports `BaseLayout` and landing sections only. It must not import `Header` or `Footer` directly.
- `src/components/Hero.astro`, `Features.astro`, `Pricing.astro`, `FAQ.astro`, and `CTA.astro` are intentional foundation stubs. Do not turn them into final marketing design in this phase.
- Shared placeholder section markup belongs in `src/components/PlaceholderSection.astro`.
- `src/components/ArticleSearch.astro` is the shared article search UI. Use `variant="header"` in `Header.astro` and `variant="page"` on `/articles/`; do not duplicate search markup or filtering logic elsewhere.
- `src/components/Header.astro` follows the KnitTools locale model: render only the current route language, link Articles/Artikkelit directly to that language's index, and do not add a visible language-switch link or restore the old article mega-menu.
- `src/components/ArticleIndexPage.astro` owns the shared English/Finnish article-index composition. It renders the localized hero copy, search, category jump links, three article previews per hub, localized remaining-count links, and the index CTA. Keep the language route files thin and do not duplicate this structure.
- `src/components/ArticleCTA.astro` is the compact product CTA for article routes. Use it after the content on English and Finnish hub/detail pages; pass `locale="fi"` on Finnish routes and `includeGuidesLink={false}` on language index pages so they do not link to themselves.

## Content

- Articles use the Astro content collection in `src/content.config.ts`.
- Article Markdown files live under `src/content/articles`.
- Article frontmatter schema:
  - `title: string`
  - `description: string`
  - `listSummary?: string`
  - `hub: "battery" | "charging" | "thermal" | "network" | "storage" | "performance" | "hardware" | "software" | "physical-damage" | "device-health" | "buying-selling" | "myths" | "brands"`
  - `sourceNumber: number`
  - `order: number`
  - `subgroup?: string`
  - `tags: string[]`
- `locale: "en" | "fi" | "sv" | "nb" | "de" | "da"` (defaults to `"en"` for existing English content)
  - `draft: boolean`
- `src/data/articleTaxonomy.mjs` owns the article hub menu and hub ordering.
- `src/data/articleTaxonomy.fi.mjs` owns Finnish hub titles, descriptions, and localized menu group labels while reusing the English taxonomy's source-number assignments.
- `src/pages/articles/index.astro` and `src/pages/fi/artikkelit/index.astro` filter their locale's non-draft entries and pass them to `ArticleIndexPage.astro` with the matching taxonomy.
- `src/pages/articles/[hub].astro` renders hub pages; `src/pages/articles/[hub]/[...slug].astro` renders individual article pages.
- `src/pages/fi/artikkelit/` mirrors the article index, hub, detail, and search-index routes in Finnish. Keep user-facing route segments and filenames Finnish, while frontmatter `hub` values remain the shared internal English keys.

## Verification

- Use `npm run build` as the foundation gate.
- Verify `/`, `/articles/`, `/fi/artikkelit/`, representative English and Finnish hub/detail pages, both article search indexes, `/robots.txt`, reciprocal `hreflang`, and language-correct canonical/HTML/OG metadata after structural changes.
