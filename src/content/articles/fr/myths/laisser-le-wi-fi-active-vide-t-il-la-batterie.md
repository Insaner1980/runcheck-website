La plupart des personnes se trompent de problème avec le Wi-Fi. Un téléphone connecté à un réseau Wi-Fi puissant n'est pas dans la même situation qu'un téléphone qui passe la journée hors connexion à rechercher des réseaux disponibles.

Cette différence compte. Une connexion Wi-Fi stable consomme généralement peu. La recherche constante de réseaux peut davantage peser sur la batterie.

## Une connexion Wi-Fi stable pose rarement problème

Lorsque votre téléphone est connecté à un routeur stable et transfère peu de données, la puce Wi-Fi passe beaucoup de temps dans des états de faible consommation. Elle se réveille pour traiter du trafic, puis se remet au repos. Comparé à l'écran, aux jeux, à l'appareil photo, à la navigation GPS ou à un mauvais signal mobile, le Wi-Fi inactif représente généralement une petite part de la consommation.

C'est pourquoi couper le Wi-Fi à la maison économise souvent moins de batterie qu'on ne l'imagine. Cela peut même pousser les applications à utiliser les données mobiles, qui consomment parfois davantage lorsque le signal du réseau mobile est faible.

La règle la plus simple est la suivante : si vous disposez d'un bon Wi-Fi, utilisez-le.

## La recherche de réseaux est différente

La situation change lorsque le téléphone n'est pas connecté. Android peut rechercher les réseaux à proximité afin de savoir lesquels sont disponibles. Les fonctions de localisation peuvent aussi utiliser les points d'accès Wi-Fi voisins pour améliorer la position, même si vous ne cherchez pas à rejoindre un réseau.

Sous Android 12 et les versions ultérieures, le réglage utile se trouve généralement dans **Paramètres > Localisation > Services de localisation > Recherche Wi-Fi**. Les fabricants déplacent parfois les menus, mais il s'agit du chemin de l'interface Android de Google. La désactivation de cette option n'empêche pas le Wi-Fi normal de fonctionner. Elle arrête les recherches Wi-Fi utilisées pour améliorer la localisation.

Android limite aussi la fréquence des recherches Wi-Fi lancées par les applications. Une application quelconque en arrière-plan ne peut donc plus rechercher les réseaux aussi agressivement que sur les anciens téléphones Android. Les services système, les réglages de localisation et les applications très actives sur le réseau peuvent malgré tout faire la différence entre un Wi-Fi au repos et un Wi-Fi réellement sollicité.

C'est de là que vient l'ancien conseil. Il y a plusieurs années, garder le Wi-Fi activé en marchant dans une ville pouvait coûter davantage de batterie. Android gère mieux la situation aujourd'hui, mais les principes physiques n'ont pas changé : une radio qui recherche activement des réseaux consomme plus qu'une radio en veille.

## Wi-Fi ou données mobiles

Pour un même téléchargement, le Wi-Fi est généralement plus économe. La distance est plus courte, la puissance radio plus faible et le transfert se termine vite. Le téléphone peut ensuite se remettre au repos.

Le réseau mobile est plus complexe. Les radios LTE et 5G maintiennent des états réseau plus élaborés et peuvent rester brièvement dans un mode de forte consommation après un transfert, au cas où d'autres données arriveraient. Un signal faible aggrave la situation. Si le téléphone peine à joindre une antenne, le modem doit travailler davantage.

Il existe une exception évidente : un mauvais Wi-Fi. Si le téléphone s'accroche à un routeur faible situé deux étages plus loin, perd sans cesse la connexion ou bascule entre le Wi-Fi et les données mobiles, un réseau mobile puissant peut être préférable.

Aucun réglage n'est le bon dans toutes les situations.

## Ce qui consomme réellement la batterie

L'écran est généralement le principal poste de consommation. Examinez ensuite les applications, la qualité du signal, la localisation, l'appareil photo, les jeux, les appels vidéo, le partage de connexion et la synchronisation en arrière-plan.

Le Wi-Fi mérite rarement d'être le premier suspect, sauf si quelque chose de précis se produit. Une sauvegarde dans le cloud peut être en cours. Une application de messagerie peut être bloquée en synchronisation. Un test de débit peut se répéter. Une application qui utilise beaucoup la localisation peut réveiller le téléphone trop souvent. Dans tous ces cas, le Wi-Fi n'est que le canal. C'est l'application qui consomme l'énergie.

L'écran **Utilisation de la batterie** d'Android est un meilleur point de départ. Si une application se trouve très haut dans la liste, limitez son activité en arrière-plan ou désinstallez-la. Couper le Wi-Fi ne corrigera pas une mauvaise application. Cela changera seulement la radio qu'elle utilise.

## Quand désactiver le Wi-Fi

Désactivez le Wi-Fi si vous restez longtemps loin de tout réseau connu, par exemple pendant une randonnée, un trajet dans une zone rurale ou une journée de travail sans Wi-Fi utilisable. Cela peut aussi aider si le téléphone est connecté à un réseau défaillant qui coupe sans cesse.

Vous pouvez également désactiver la recherche Wi-Fi liée à la localisation si vous n'en avez pas besoin. C'est une solution plus ciblée que de couper complètement le Wi-Fi, car la connexion réseau normale reste disponible.

La nuit, la différence est généralement faible. Si vous n'avez pas besoin des notifications, le mode Ne pas déranger, le mode Coucher ou l'Économiseur de batterie sont souvent plus adaptés. Le mode Doze d'Android limite déjà le réseau en arrière-plan et l'activité du CPU lorsque le téléphone est inactif, que l'écran est éteint et que l'appareil est débranché.

## Quelques cas particuliers

Les réseaux Wi-Fi enregistrés ne consomment pas de batterie par eux-mêmes. Une longue liste de réseaux connus ne signifie pas que le téléphone les vérifie tous en permanence dans une boucle coûteuse.

Les appels Wi-Fi peuvent consommer moins que les appels mobiles lorsque le signal mobile est faible. Si les signaux Wi-Fi et mobile sont tous deux puissants, la différence ne mérite généralement pas qu'on s'en préoccupe.

Pour la plupart des personnes, laisser le Wi-Fi activé ne pose aucun problème. Ce qu'il faut éviter, ce sont les signaux faibles, les recherches constantes et les applications qui réveillent sans cesse le téléphone.
